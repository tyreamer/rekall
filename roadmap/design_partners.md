# Design Partners (v0.1)

This doc tracks outreach targets and what we’re validating.

---

## 1) Ideal design partner profiles

### A) Agent-heavy solo builders
- Uses Claude Code / Cursor / local agents daily
- Has multiple side projects and struggles with context + resumability
- Will adopt local-first tools without accounts

### B) Small teams experimenting with agents
- 2–6 people
- Shared work causes duplication and “what did we try?” confusion
- Needs lightweight permissions and audit trail

### C) Enterprise platform / delivery leads (roadmap sanity)
- Director/manager who wants reliable status without status-meeting overhead
- Security-minded architects who care about secrets + audit

---

## 2) What we validate (POC)

### Primary hypotheses
- H1: A portable state artifact materially reduces “context paste” overhead
- H2: Attempt + decision logs prevent repeated dead ends (“anti-loop” value)
- H3: Exec queries via chat are trusted when backed by evidence links
- H4: Claim/lease reduces duplicate agent work and increases confidence

### Success signals
- Users keep state updated because it saves them time
- They ask for collaboration/hub after using local-first
- They share the spec with others (“this should be standard”)

---

## 3) Interview script (15–25 minutes)

1) Show: sample state artifact for a dormant project
2) Ask: “What’s missing for you to resume in 30 minutes?”
3) Ask: “What do you paste repeatedly into agents today?”
4) Ask: “What has been tried already that you forget / redo?”
5) Ask: “If a teammate joined, where do they get truth?”
6) Ask: “What should an exec status answer include to be trusted?”
7) Ask: “Would you run this locally? Would you pay for hosted collaboration?”

---

## 4) Pilot tasks (hands-on)

- Create a new project with state artifact template
- Add 5 work items + 2 attempts + 1 decision
- Ask the exec questions Q1–Q10 and rate trustworthiness
- Resume after 7 days and measure time-to-context
