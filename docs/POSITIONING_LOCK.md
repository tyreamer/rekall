# Positioning Lock

**Category One-Liner (Must appear in first 15 words of README and landing pages):**
> Rekall is the verifiable, replayable AI execution record and audit trail for autonomous work.

## Approved Phrases
Use only these or direct variants:
- verifiable AI execution record
- deterministic execution audit trail
- tamper-evident replay substrate
- evidence-backed status for AI agents
- local-first, git-portable append-only execution ledger

## Consensus Primitives (The Evidence)
A Rekall execution record is composed strictly of:
- **`ActionProposed`**: Cryptographically hashed intent.
- **`PolicyCheck`**: Automated shadow audit against the execution policy.
- **`Decision`**: Result of a human or higher-order resolver wait.
- **`HumanAnchor`**: Proof-of-Human signature (HMAC-SHA256).
- **`Outcome`**: Verifiable proof of execution success or failure.

## Banned Terms
Grep must return zero matches in all `.md` files and CLI help text for:
- memory (persistent, agent, long-term, project, etc.)
- blackboard
- task tracker / kanban / board
- project state for humans
- ledger (unless prefixed as “execution ledger”)

| IS | IS NOT |

| evidence-backed status for AI agents | project memory |
| local-first git-portable append-only execution ledger | long-term memory |
